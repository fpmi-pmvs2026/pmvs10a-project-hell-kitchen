# MealHunter — Презентация проекта

## Обзор

**MealHunter** — Android-приложение для поиска, фильтрации и сохранения рецептов.  
Технологии: Kotlin + Jetpack Compose + Material 3 + Spoonacular API.

---

## Проблема

Поиск новых рецептов неудобен: нужно открывать браузер, пролистывать рекламу, вручную выписывать ингредиенты. Не существует быстрого и увлекательного способа решить, что приготовить из того, что уже есть в холодильнике.

## Решение

MealHunter предоставляет:
- **Свайп-просмотр** — листаем рецепты как колоду карточек
- **Умный поиск** — по названию, кухне, диете или ингредиентам из холодильника
- **Офлайн-избранное** — сохранённые рецепты доступны без интернета

---

## Экраны приложения

| Экран | Описание |
|-------|---------|
| **Discover (Главный)** | Случайные рецепты-карточки. Свайп вправо — сохранить, влево — пропустить. Фильтры по кухне, диете, типу, времени |
| **Search (Поиск)** | Текстовый поиск с историей запросов. Результаты — прокручиваемый список |
| **Detail (Детали)** | Фото, время, порции, health score, список ингредиентов с чекбоксами, пошаговые инструкции, кнопка избранного |
| **Favorites (Избранное)** | Сохранённые рецепты. Удаление свайпом влево |
| **My Fridge (Холодильник)** | Добавляем ингредиенты чипами, ищем подходящие рецепты |

## API

Используется [Spoonacular Food API](https://spoonacular.com/food-api):
- `GET /recipes/complexSearch` — поиск с фильтрами
- `GET /recipes/random` — случайные рецепты для главного экрана
- `GET /recipes/{id}/information` — полные данные рецепта

API-ключ хранится в `gradle.properties` (не коммитится) и GitHub Secrets, внедряется через `BuildConfig`.
